================================================================================
CORE PHILOSOPHY
================================================================================

1. SURGICAL, NOT SWEEPING
   - Read and understand every existing file before touching anything.
   - Make the minimum viable change that produces maximum visual impact.
   - Never rewrite a working system. Extend and enhance it.
   - Preserve the architectural decisions already made unless they are
     fundamentally broken.

2. BRAND-FIRST DESIGN
   - Extract the logo's exact colour palette and make it the DNA of the entire
     site — backgrounds, glows, gradients, borders, hover states, and accents.
   - Every design decision must answer: "Does this reinforce the brand?"
   - Consistency across all pages, sections, and states is non-negotiable.

3. PREMIUM AESTHETICS WITHOUT HEAVY LIBRARIES
   - Achieve world-class visual quality using pure CSS animations, keyframes,
     and carefully crafted inline styles.
   - Avoid heavy animation libraries (Framer Motion, GSAP) unless the project
     already uses them. CSS transitions are faster and lighter.
   - Glass morphism, neon glows, gradient text, shimmer effects, and ambient
     orbs create premium feel with zero bundle cost.

4. PERFORMANCE AS A DESIGN PRINCIPLE
   - Cloudflare Workers + edge caching is the gold standard deployment model.
   - Preconnect to all external font/icon CDNs before loading them.
   - Use `display=swap` on all Google Fonts.
   - Keep JavaScript minimal; let CSS carry the visual weight.

COLOUR PALETTE INTEGRATION PROTOCOL
================================================================================

When given a logo or brand colour palette, execute the following:

STEP 1 — EXTRACT
  - Identify the PRIMARY accent colour (the boldest/most distinctive hue)
  - Identify the SECONDARY accent colour (supporting hue or complementary)
  - Identify the BASE DARK colour (background anchor)
  - Identify the TEXT HIGHLIGHT colour (light version of primary for text)

STEP 2 — MAP TO SITE ANATOMY
  Background base:      darkest brand colour or near-black variant (#030712 pattern)
  Hero gradient:        primary → secondary diagonal or radial gradient
  Section orbs/glows:   primary colour at 20-40% opacity, blurred
  CTA buttons:          primary → secondary gradient with shimmer animation
  Active nav links:     primary accent with bottom border or glow
  Card hover borders:   primary accent at 60% opacity
  Gradient text:        light variant of primary → white
  Progress bar:         primary → secondary gradient
  Logo text glow:       primary at 40% opacity drop-shadow

STEP 3 — THEME VARIABLES
  Define a theme object (e.g., in src/utills/themes.js) with:
  {
    primary, secondary, accent,
    gradient, gradientReverse,
    glowColor, glowSoft,
    textGradientFrom, textGradientTo,
    cardBorder, navActive, buttonBg
  }

STEP 4 — PROPAGATE
  Apply theme variables consistently across:
  - App.js (header, footer, nav)
  - All page hero sections
  - SectionWrapper glow orbs
  - All CTA/button elements
  - All card hover states

STANDARD KEYFRAME SET (add to public/index.html <style> block):

    from { opacity: 0; transform: translateY(30px); }
    to   { opacity: 1; transform: translateY(0); }
  }

  @keyframes fade-in-left {
    from { opacity: 0; transform: translateX(-30px); }
    to   { opacity: 1; transform: translateX(0); }

    from { opacity: 0; transform: translateX(30px); }
    to   { opacity: 1; transform: translateX(0); }
  }

    from { opacity: 0; transform: scale(0.9); }
    to   { opacity: 1; transform: scale(1); }

CLOUDFLARE WORKERS DEPLOYMENT

WRANGLER.TOML PATTERN:
  name = "site-name"
  main = "worker/index.js"
  compatibility_date = "YYYY-MM-DD"  // use current date
  [site]
  bucket = "./build"

WORKER/INDEX.JS PATTERN (SPA routing support):
  import { getAssetFromKV, NotFoundError } from '@cloudflare/kv-asset-handler';
  addEventListener('fetch', event => event.respondWith(handleEvent(event)));
  async function handleEvent(event) {
    try {
      return await getAssetFromKV(event);
    } catch (e) {
      if (e instanceof NotFoundError) {
        // Serve index.html for all SPA routes
        const url = new URL(event.request.url);
        url.pathname = '/index.html';
        return await getAssetFromKV(event, { mapRequestToAsset: req =>
          new Request(url.toString(), req) });
      }
      throw e;
    }
  }

DEPLOYMENT COMMANDS:
  npm run build && npx wrangler deploy

COMMON BUILD ISSUES:
  - package-lock.json must be in sync (run npm install before commit)
  - lock files from node_modules/ should not be committed
  - Tailwind via CDN: no PostCSS build step needed
  - Font Awesome via CDN: no build step needed
  - Ensure wrangler version compatibility with node version

COMMON SURGICAL FIXES CHECKLIST
================================================================================

When inheriting a pre-existing repository, audit and fix these in order:

CRITICAL (breaks site):
  [ ] SPA routing 404s — ensure Cloudflare Worker serves index.html for all routes
  [ ] Broken font loading — check preconnect links exist and are ordered correctly
  [ ] Missing favicon — add favicon.ico to public/
  [ ] Lock file mismatch — run npm install to regenerate package-lock.json

HIGH IMPACT (user experience):
  [ ] Mobile nav doesn't close on route change — add useEffect with location
  [ ] Animations flash (show then animate) — add opacity:0 to animated elements
  [ ] Hero text not responsive — implement clamp() fluid sizing
  [ ] Cards have inconsistent hover states — standardise hover: lift + glow pattern
  [ ] No scroll progress indicator — add fixed top bar with scroll-driven scaleX

BRANDING:
  [ ] Colors not derived from logo palette — extract hex codes, propagate everywhere
  [ ] Font mixing (more than 3 families) — consolidate to display/body/accent trio
  [ ] Inconsistent section spacing — standardise py-20 md:py-32 sections
  [ ] Footer looks like an afterthought — add logo, social icons, subtle gradient border

SEO:
  [ ] Missing JSON-LD structured data — add Schema.org markup
  [ ] Generic page title — personalise with name + role + tagline
  [ ] Missing OG image — create 1200×630 cover image
  [ ] No canonical link — add <link rel="canonical">
  [ ] Missing twitter:card meta — add full Twitter Card set

PERFORMANCE:
  [ ] Fonts not preconnected — add preconnect before stylesheet link
  [ ] No font-display:swap — add to Google Fonts URL
  [ ] CDN links not using HTTPS — ensure all external links use HTTPS
  [ ] Unused heavy dependencies — audit package.json, remove bloat

ACCESSIBILITY:
  [ ] Icon-only buttons missing aria-label — add descriptive labels
  [ ] Images missing alt text — add meaningful descriptions
  [ ] No focus visible styles — ensure :focus-visible outline exists
  [ ] Low contrast text — check against WCAG AA (4.5:1 ratio minimum)
